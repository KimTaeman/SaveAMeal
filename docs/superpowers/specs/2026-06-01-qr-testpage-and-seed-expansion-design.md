# QR Test Page & Seed Data Expansion — Design

**Date:** 2026-06-01
**Status:** Approved
**Author:** KimTaeman

## Goal

Produce two deliverables for on-device testing of all active role flows:
1. A self-contained HTML page of scannable codes (food barcodes for Donor, batch QR codes for Driver).
2. An expanded `tools/seed/seed.js` with realistic Thai food rescue data covering every batch status.

## Scanning flows per role

| Role | Scanner accepts | What to scan | Test artifact |
|------|----------------|--------------|---------------|
| Donor | Any format (`BarcodeFormat.all`) | Food product EAN-13 barcode | Section A of HTML page |
| Driver | QR code only | Batch ID QR (`batchId == activeBatch.id`) | Section B of HTML page |
| Volunteer | Not yet implemented | — | — |
| Beneficiary | No scanning | — | — |

---

## Deliverable 1 — `docs/test-qr/index.html`

Single self-contained HTML file. No build step. Uses CDN libraries:
- **JsBarcode** (`jsbarcode.all.min.js`) — renders EAN-13 barcodes as `<svg>`.
- **qrcode.js** (`qrcode.min.js`) — renders QR codes as `<canvas>`.

### Section A — Food Barcodes (Donor)

Six real Thai food products with valid EAN-13 codes. Each card shows:
- Product name + brand
- EAN-13 number below the barcode
- "Donor: Log Surplus" label

| Product | EAN-13 |
|---------|--------|
| Mama Tom Yum noodles | 8850987130022 |
| Lays Classic (Thai) | 8850018110061 |
| Oishi Green Tea | 8850007107012 |
| Pocky Chocolate | 4901005508032 |
| Milo UHT | 9556000021107 |
| Bear Brand Sterilised Milk | 7613034426826 |

### Section B — Batch QR Codes (Driver)

One QR card per claimed/open batch. Each card shows:
- Batch ID (printed below the QR)
- Donor name + status label
- "Driver: Scan at Pickup Verify" label

Batches included: all `claimed` and `open` batches from seed data (IDs generated by seed expansion, see below).

### Layout

Two-column card grid, white background, print-friendly. Section headers distinguish A from B. Page title: "SaveAMeal — Test Scan Codes".
